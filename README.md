# 环球网校平台运营数据看板｜作品集 Demo

一个完全独立于内部数据看板的公开作品集项目。所有数字均为固定、可复现的合成数据，不连接真实业务后台。

## 本地运行

```powershell
npm.cmd install
npm.cmd run dev
```

浏览器打开 Vite 输出的本地地址。

## 生产构建

```powershell
npm.cmd run build
npm.cmd run preview
```

## 已实现

- 桌面端与手机端响应式的一页式运营看板。
- 保留原看板的配置顺序：业务分类、运营类型、终端多选、取数范围、确定生成数据。
- 支持两条独立业务线：职教日常运营，以及青藤 AI 专题运营。
- 青藤 AI 专题页包含页面浏览与来源渗透、浏览行为、老师卡片、课程货架、转化与加微，并使用 IP 筛选。
- 保留原看板的数据顺序：数据概览、营收、活跃/新增、留存、转化/加微/私域；八大考试拆分归在营收内部。
- 特权卡从属于题库 APP：只有勾选题库 APP 时，特权卡入口和数据区块才会出现。
- Agent 同时提供左侧固定入口和顶部入口，打开同一个完整查询抽屉。
- 支持整页生成、模块重新生成、月份快捷选择、考试筛选、URL 状态和 CSV 导出入口。
- 取数范围支持按月取数和实时取数；实时模式提供具体日期及近 3 天、近 7 天、昨天、今天快捷选择。
- 数据 Agent 内置职教与青藤 AI 两套预设问答，推荐问题和关键词自由提问均可返回演示答案。
- 只查询本地合成数据的 Demo Agent，能结合当前业务模块回答推荐问题和自由输入。
- 无网络时仍可使用所有图表和查询逻辑。

## 数据隔离

本工程只复用原看板的业务信息架构，不复制或引用上级 `dashboard_app` 中的 CSV、SQL、数据源地址、密钥或接口。若后续接入真实模型，只允许增加独立的安全服务端接口，不能接入公司数据系统。
