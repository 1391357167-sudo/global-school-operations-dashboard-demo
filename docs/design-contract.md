# 环球网校平台运营数据看板 Demo 设计契约

- 审批状态：已批准（2026-07-16），用户确认单张桌面效果图方向可用于 Demo。
- 概念参考：`public/accepted-concept.png`。
- 产品边界：独立作品集产品；保留原看板业务信息架构，但不得导入内部看板的真实 CSV、SQL、密钥、接口地址或公司名称。
- 配置结构：业务分类 → 运营类型 → 终端多选 → 取数范围 → 确定生成数据。
- 业务切换：职教对应日常运营与终端筛选；青藤 AI 对应专题运营与 IP 筛选。
- 青藤 AI 数据结构：页面浏览与来源渗透 → 浏览行为（场景案例、老师卡片、课程货架）→ 专题页转化与加微。
- 数据结构：数据概览 → 营收（流量营收、终端营收、八大考试营收拆分）→ 活跃/新增（新增活跃、场景活跃）→ 留存 → 转化/加微/私域。
- 从属关系：特权卡属于题库 APP，只有选择题库 APP 时才显示。
- 主阅读路径：先配置并生成数据，再沿原页面的一页式锚点顺序阅读；Agent 作为辅助查询入口。
- 视觉基准：真白背景、深海军蓝侧栏、电光蓝主色、绿色正向、橙色预警、轻边框、少阴影、6–8px 圆角。
- 锁定元素：深色侧栏、顶部横向筛选、蓝色洞察条、四个 KPI、单一主趋势图、右侧 Agent、漏斗与表格并列。
- 灵活元素：脱敏后的业务文案、合成数字、移动端断点、图表刻度密度；不得凭空增加招生、教学或校区等原看板不存在的业务模块。
- 数据状态：全部为固定、可复现的合成演示数据；页面底部和 Agent 结果中明确标识。
- 图表实现：趋势图和微型图使用 SVG；排行、漏斗、表格图形使用 CSS/HTML；首屏约 7 个证据视图，无 WebGL。
- 交互：筛选写入 URL；刷新保留最后可见数据；CSV 导出；Agent 只调用预定义的本地查询逻辑。
- 取数模式：按月显示月份范围与月度快捷选项；实时显示日期范围与近 3 天、近 7 天、昨天、今天。
- Agent：按当前业务切换推荐问题库；点击推荐问题直接回答，自由输入按关键词命中预设答案，并明确标识为 Demo 预设问答。
- 移动端：隐藏侧栏，KPI 横向滚动，趋势图优先，筛选与 Agent 使用抽屉，保留 44px 左右触控区域。
- 降级：无网络仍可查看和查询全部假数据；减少动态效果时保留所有结论和数值。

## 可视化清单

| 视图 | 任务 | 形式 | QA |
| --- | --- | --- | --- |
| KPI | 快速比较当前值与变化 | 数字 + SVG sparkline | 数值与筛选同步 |
| 月度趋势 | 识别时间变化与业务峰值 | SVG 折线/面积 + 标注 | 峰值和末期变化可见 |
| 业务排行 | 比较终端、考试或场景贡献 | 横向条形图 | 排序及长度一致 |
| 转化漏斗 | 展示阶段流失 | CSS 漏斗 | 阶段值单调下降 |
| 明细表格 | 查询与比较 | 表格 + 微型柱 | 排名、涨跌色义一致 |
| Agent | 解释筛选范围内的证据 | 文本 + 证据条 | 不生成真实数据声明 |
